stop slop a writing skill.

a public writing skill for removing common ai patterns from prose.

use it to audit drafts, rewrite content, or pressure-test writing that feels too polished, too generic, or too obviously machine-generated.

§ 01

what it does.

a short, opinionated editing pass — not a voice layer.

  1. 01

    flags common ai writing patterns

    word-level hits, structural tells, and voice-level stance problems, all flagged separately.

  2. 02

    supports two modes

    rewrite detect — default is a rewrite with a second pass; detect is flag-only.

  3. 03

    splits word-level from structural

    vocabulary tiers, pattern tells, and formatting artifacts live in three separate references.

  4. 04

    encourages a second pass

    one clean-up rarely fixes rhythm. the skill rewrites, then pressure-tests its own rewrite.

  5. 05

    works as a local skill or a prompt pack

    drop it into claude code or codex. paste it into a chatgpt project. use it in one-off threads.

§ 02

two modes.

default is rewrite. switch to detect when you want flags without prose.

default

rewrite

  • flag the main issues
  • rewrite the text
  • run a second pass on the rewrite
output issues · rewrite · what changed · second pass
on request

detect

  • flag only
  • show what reads as ai
  • separate clear problems from judgment calls
output issues · assessment · fix direction

trigger detect with detect, flag only, audit only, or scan.

    § 05

    word flags, by tier.

    editing pressure points, not absolute bans. context still decides.

    tier 1

    replace on sight

    unless there is a clear reason not to.

    tier 2

    flag in clusters

    two or more in one paragraph is usually a bad sign.

    tier 3

    flag at density

    normal words. only a problem when the draft leans on them too often.

    § 07

    severity.

    three bands. credibility first, then smell, then polish.

    1. p0

      credibility

      • fake sourcing
      • chatbot artifacts
      • cutoff disclaimers
      • markdown in plain-text contexts
      • obvious significance inflation
    2. p1

      clear ai smell

      • tier-1 word hits
      • repeated templates
      • formulaic openings
      • repeated contrasts
      • rhythm that feels machine-even
    3. p2

      polish

      • too many triads
      • too many bullets
      • repetitive paragraph shapes
      • conclusion that says nothing

    be stricter in

    • essays
    • investor emails
    • public posts
    • website copy

    be looser in

    • docs
    • technical notes
    • fast internal chat
    § 08

    install.

    local skill, prompt pack, or paste-in. pick the one that matches where you write.

    # ~/.claude/skills/stop-slop
mkdir -p ~/.claude/skills
cp -R stop-slop-public ~/.claude/skills/stop-slop
    # ~/.codex/skills/stop-slop
mkdir -p ~/.codex/skills
cp -R stop-slop-public ~/.codex/skills/stop-slop

    use SKILL.md as the main instruction file. pull in files under references/ for the word list, the pattern list, or examples.

    # attach as project files
SKILL.md
references/words.md
references/patterns.md
references/examples.md

    paste the key parts of SKILL.md into a project instruction, custom instruction, or the first message of a thread.

    # trigger phrases
rewrite this with stop slop
audit this for ai writing patterns
detect only. do not rewrite
use stop slop on this memo
